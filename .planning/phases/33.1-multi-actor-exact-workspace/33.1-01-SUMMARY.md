# Phase 33.1 Plan 01 Summary

**The browser demo now holds an imported workpiece and a generated tool in one actor-scoped workspace, and the tool can move through one authoritative store-owned pose contract.**

## Accomplishments

- Added `demo/src/lib/workspace-model.js` to derive a composite display model from canonical workspace actors, preserving actor identity while keeping the existing Babylon viewer surface intact.
- Reworked `viewerStore` so:
  - workpiece and tool are stored as separate actor records
  - orientation mode remains workpiece-specific
  - tool pose is owned by store state through `setActorPose(...)` and `nudgeActorPose(...)`
  - selection state is cleared whenever actor topology or actor pose changes
- Reworked `useOcct` so importing CAD replaces only the workpiece actor and generating a revolved tool replaces only the tool actor, while exact-session disposal stays role-scoped.
- Added an explicit tool-pose panel to `SelectionPanel` with deterministic pose nudges for the generated tool actor.
- Strengthened demo regression coverage:
  - store tests now lock actor coexistence, exact-session separation, and tool-pose updates
  - runtime-hook contract tests now lock actor-scoped import/generation wiring
  - Playwright now verifies imported workpiece + generated tool coexistence and tool-pose movement
- Hardened browser harness reliability by isolating Playwright onto a dedicated local Vite port instead of reusing arbitrary existing `5173` servers.
