<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| actor-scoped workspace -> retained exact lifecycle | If actor replacement or reset does not dispose the right exact session, later measurements will inherit stale handles or leaked actors |
| store-owned pose -> Babylon render path | If tool pose lives only in Babylon scene nodes, exact refs and later measurements will diverge from what the user sees |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33.1-01-01 | T/R | actor replacement and reset | mitigate | Add store/runtime/browser tests before implementation and require explicit dispose-all semantics on workspace reset |
| T-33.1-01-02 | I | render pose drift | mitigate | Make actor pose authoritative in store state and have the viewer consume that pose rather than inventing Babylon-local transforms |
</threat_model>
