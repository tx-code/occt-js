# Phase 33.1 Plan 02 Summary

**The root/package exact surface now supports additive cross-model pairwise measurement, and the demo selection pipeline emits actor-scoped exact refs that are ready for workpiece-plus-tool measurement flows.**

## Accomplishments

- Added explicit root `CrossModel` pairwise entrypoints for:
  - exact distance
  - exact angle
  - exact thickness
  - exact relation classification
  - exact distance/angle/thickness placements
- Kept the original same-model root entrypoints source-compatible by forwarding them through the new shared cross-model implementation path.
- Updated the tracked root typing contract in [dist/occt-js.d.ts](/E:/Coding/occt-js/dist/occt-js.d.ts:683) so downstream consumers can see the additive `*CrossModel(...)` APIs.
- Reworked `packages/occt-core` pairwise wrappers to dispatch automatically:
  - same `exactModelId` uses the legacy root methods
  - different `exactModelId` uses the additive `*CrossModel(...)` root methods
- Rebuilt the demo selection bridge around actor-scoped exact resolution:
  - picks now recover actor identity from workspace-prefixed `nodeId`/`geometryId`
  - picks resolve through `resolveExactElementRef(...)` against the selected actor's exact model
  - stored `exactRef.transform` now composes `actorPose * occurrenceTransform`
  - selection state now carries `actorId`, actor-local ids, and exact refs instead of only Babylon mesh handles
- Normalized demo exact-session payloads at creation time so the bridge operates on stable normalized exact-open data rather than raw root payloads.
- Hardened browser verification by adding a focused bridge test suite and by switching two flaky center-click Playwright checks to the same projected-visible-point strategy already used elsewhere.
