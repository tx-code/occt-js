| Boundary | Description |
|----------|-------------|
| caller-supplied extruded spec -> shared profile kernel + family validator | If extrusion-specific depth/unit rules are not layered cleanly over the shared kernel, callers may validate invalid geometry or inherit family-specific behavior in the wrong place |
| exact OCCT prism build -> generated-scene payload | If prism construction or triangulation drifts from canonical generated-family behavior, downstream scene consumers will observe a different payload shape or broken exact-open lane |

| T-31-01-02 | T/I | build/openExact family adapter | mitigate | Reuse the existing generated-family build/openExact pattern and keep shared-profile + revolved-family drift guards in the quick/full verification commands |
</threat_model>
