# Milestone v1.12: Surface Subtraction & Boundary Cleanup

**Status:** ✅ SHIPPED 2026-04-21
**Phases:** 37-39
**Total Plans:** 6

## Overview

`v1.12` was an intentional subtraction milestone. Instead of widening the measurement or generated-shape surface again, it pulled `occt-js` back toward its actual runtime/package boundary: the browser demo was simplified into a clearer integration sample, demo-owned policy drift was pruned away from package-adjacent seams, and live docs plus conditional verification were realigned to the reduced surface without widening the authoritative root release gate.

## Phases

### Phase 37: Demo Simplification & Sample-First Workflow

**Goal**: The browser demo shows one clear integration workflow and sheds product-style measurement/tool workflow complexity that does not prove runtime/package value.
**Depends on**: Phase 36
**Plans**: 2 plans

Plans:

- [x] 37-01-PLAN.md — Simplify measurement/action presentation to an explicit sample-first workflow
- [x] 37-02-PLAN.md — Remove unnecessary demo workflow state and tighten tool/workpiece interaction surfaces

**Details:**
- The measurement inspector now presents explicit supported actions and one current-result surface instead of a product-style result history.
- Demo measurement state was reduced to one `currentMeasurement`, and the browser workflow now reads as import/generate, select, run, inspect.
- Entry copy and workflow emphasis now consistently push the import-first/sample-first path instead of a half-product tool workspace.

### Phase 38: Boundary & API Pruning

**Goal**: Root/package/demo boundaries are cleaned up so app-owned policy no longer drifts into retained SDK or helper-adjacent surfaces.
**Depends on**: Phase 37
**Plans**: 2 plans

Plans:

- [x] 38-01-PLAN.md — Audit and prune app-owned measurement integration surfaces from package/runtime-adjacent code
- [x] 38-02-PLAN.md — Tighten typings, exports, and helper ownership around the reduced contract

**Details:**

- The package barrel/types stayed unchanged because the audit showed they were already minimal; cleanup landed through naming, tests, and one minimal wording fix.

### Phase 39: Docs & Verification Realignment

**Goal**: Docs, contract tests, and conditional verification reflect the simplified surface and continue to protect the root release boundary.
**Depends on**: Phase 38
**Plans**: 2 plans

## Milestone Summary

**Key Decisions:**
- Use `v1.12` as a subtraction milestone before considering more feature growth, because the repo needed simplification and boundary cleanup more than additional behavior.
- Keep measurement action mapping and session UX demo-owned instead of reopening package-owned candidate-analysis or session workflows.
- Prefer stronger contract tests and minimal wording fixes over gratuitous API churn when package/runtime surfaces are already sufficiently narrow.

**Issues Resolved:**
- Reduced the browser demo from a product-like measurement workspace back to an explicit integration sample.
- Removed the remaining demo-owned action/session drift from package-adjacent naming, docs, and contract expectations.
- Realigned live docs, planning truth, and conditional verification so the reduced surface is enforced consistently.

**Issues Deferred:**
- `SEED-001-web-exact-brep-measurement` remains partially resolved and dormant.
- Any future package-owned action analysis or persistent measurement session workflow must be scoped in a fresh milestone instead of being implicitly carried forward.

**Technical Debt Incurred:**
- `v1.12` closed without a dedicated milestone-audit artifact; closeout relies on manual milestone review, `audit-open`, contract evidence, and `npm run test:planning:audit`.
