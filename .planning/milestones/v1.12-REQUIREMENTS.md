# Requirements: occt-js

**Defined:** 2026-04-21
**Core Value:** Downstream applications can reliably consume the OCCT Wasm runtime and its root API contract without build drift or packaging surprises.

## v1 Requirements

### Demo Simplification

- [x] **DEMO-05**: The browser demo exposes a smaller, explicit integration workflow for import, tool/workpiece setup, selection, and measurement instead of a product-style productivity surface.
- [x] **UX-04**: Demo measurement UI only presents explicit supported actions and removes unnecessary states or controls that do not prove runtime/package integration value.

### Boundary Cleanup

- [x] **BOUND-01**: Package/runtime-adjacent code and docs do not expose app-owned candidate, session, or similar policy-bearing behavior as part of the retained public contract.
- [x] **SDK-04**: `@tx-code/occt-core` and demo-owned integration helpers keep measurement action mapping and session semantics downstream with clear ownership boundaries.

### Docs & Governance

- [x] **DOCS-07**: Root, package, and demo docs describe the simplified integration sample and reduced measurement surface without implying a richer measurement product workflow.
- [x] **GOV-05**: Contract tests and conditional verification fail on drift in the reduced surface while keeping `npm run test:release:root` unchanged.

## v2 Requirements

### Deferred Product Surface

- **FEAT-06**: Demo users or downstream apps can derive supported measurement actions from the current actor-scoped exact workspace without falling back to mesh heuristics.
- **UX-03**: Demo users can pin, compare, clear, or export measurement sessions beyond the current transient history without stale-handle or actor-pose drift.
- **FEAT-09**: Package-owned action analysis or richer helper guidance can be reconsidered only if a concrete reusable downstream gap is proven.

## Out of Scope

| Feature | Reason |
|---------|--------|
| New exact measurement primitives or new semantic helper families | `v1.12` was subtractive and boundary-cleanup-focused, not another capability-expansion milestone |
| Package-owned candidate-analysis APIs or session workflows | Action mapping and session behavior remain downstream app concerns |
| Full measurement-product UX such as persistent history, reporting, tolerances, or collaboration | The repository should stay runtime/package-first with a demo integration sample |
| Desktop-first simplification work that changes root publishing assumptions | Web/runtime/package boundaries remain the primary contract |
| Broad release-pipeline rewrites unrelated to surface subtraction | The milestone stayed focused on boundary cleanup, not stable release-governance redesign |
