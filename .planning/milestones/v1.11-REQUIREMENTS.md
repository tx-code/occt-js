# Requirements: occt-js

**Defined:** 2026-04-21
**Core Value:** Downstream applications can reliably consume the OCCT Wasm runtime and its root API contract without build drift or packaging surprises.

## v1 Requirements

### Supported Semantic Helpers

- [x] **FEAT-07**: Downstream apps can use package-first semantic measurement helpers for supported families such as `counterbore` and `countersink` over the shipped exact surface.
- [x] **SDK-03**: `@tx-code/occt-core` exposes typed package-first semantic-helper APIs that preserve occurrence-safe refs, explicit unsupported results, and additive compatibility with existing exact wrappers.

## v2 Requirements

### Deferred Measurement Productivity

- **FEAT-06**: Demo users or downstream apps can derive supported measurement actions from the current actor-scoped exact workspace without falling back to mesh heuristics.
- **DEMO-04**: Demo users can trigger supported semantic-helper workflows from the current workspace while keeping selection-to-action mapping in app code instead of package-owned candidate analysis.
- **UX-03**: Demo users can pin, compare, clear, or export measurement sessions beyond the current transient history without stale-handle or actor-pose drift.
- **DOCS-06**: Root, package, and demo docs explain any future helper/action/session workflow without implying viewer widgets are part of the runtime contract.
- **E2E-02**: Automated verification covers any future action suggestion or session-productivity workflow in the demo.
- **GOV-04**: Typings, package tests, and conditional governance lock any future productivity surface without widening `npm run test:release:root`.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Package-owned candidate-analysis APIs or whole-model measurement discovery | The active helper-only scope kept selection-to-action mapping in app code and avoided reopening package/runtime ownership |
| Pinned, exportable, or persistent measurement sessions | Session persistence and reporting remain downstream product concerns, not the shipped helper-only milestone scope |
| Full AIS/Prs3d-style dimension widgets, label layout, or annotation editing | `v1.11` stayed narrow and helper-first, not a full dimension-authoring product |
| Broad new root Wasm measurement APIs without package-first justification | The root carrier must stay narrow; new runtime support must be additive and justified by composition needs |
| ML- or heuristic-driven whole-model feature recognition | The milestone must stay deterministic, supported-family-based, and exact-ref-driven |
